PreDCR GeoJSON-to-CAD Automation Suite 🏢🗺️Project OverviewThis project is a Proof of Concept (PoC) developed to bridge the gap between web-based spatial data (GeoJSON) and municipal building plan standards (AutoCAD DXF).In urban planning and architecture, drawings must adhere to strict PreDCR (Pre-Development Control Regulation) guidelines. This suite automates the conversion process, ensuring that geographic features are not just converted into shapes, but are routed to the correct CAD layers with mandatory color codes required for automated regulatory approval.Repository StructureThe project is organized into a modular architecture to separate core processing logic from test data payloads.PlaintextPreDCR/

Technical Implementation1. Strict Compliance Engine (PreDCR_comp.py)This is the primary tool for regulatory workflows. It uses a Rule-Based Mapping System to enforce municipal standards:Mandatory Layer Routing: Automatically maps feature names to strict layers (e.g., _PlotBoundary, _PropWork, _Room, _Window, _Road, _MarginalOpenSpace).Standardized Coloring: Enforces AutoCAD Color Indices (ACI), such as Magenta (6) for boundaries and Color 20 for roads.String Normalization: Cleans and normalizes input names; for example, " ROAD " or "_road" both resolve to the official _Road layer.Fallback Protocol: Unknown shapes are automatically prefixed with an underscore and assigned Color 7 (White/Black) to ensure no data is lost during conversion.2. Flexible Translation Engine (geojson_to_dxf.py)A utility script designed for general drafting where strict PreDCR rules are not required:Property Respect: Directly reads layer and color properties from the GeoJSON if provided.Dynamic Layering: If a layer name is missing, it auto-generates one based on the shape's name or a unique counter.3. Advanced Extraction (dxf_to_json_auto.py)A reverse-engineering tool that converts DXF back to GeoJSON. It features a recursive Block Processor that automatically "explodes" INSERT entities to find and extract hidden geometry tucked inside CAD blocks.Geometry Mapping TableThe following table illustrates how geographic data is translated into CAD entities to maintain structural integrity.GeoJSON FeatureCAD EntityPreDCR UsagePolygonLWPOLYLINE (Closed)Plots, Rooms, Building FootprintsLineStringLWPOLYLINE (Open)Roads, Windows, Compound WallsPointCIRCLETrees, Columns, Fire HydrantsGetting StartedPrerequisitesEnsure you have Python 3.x installed along with the ezdxf library:Bashpip install ezdxf
Running the ConversionTo generate a PreDCR-compliant DXF from a GeoJSON layout:Bashpython src/PreDCR_comp.py data/floor.geojson data/floor.dxf
To run a flexible conversion for general shapes:Bashpython src/geojson_to_dxf.py data/shapes.geojson data/output.dxf
